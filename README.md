# Unity First 2D Game

Unity로 처음 제작한 2D 게임 프로젝트입니다.  
이 프로젝트는 Unity의 기본 구조를 익히고, 직접 간단한 게임을 완성해보는 것을 목표로 진행했습니다.

플레이어는 좌우 이동과 점프를 통해 장애물을 피하고, 코인을 모아 목표 점수에 도달하면 게임을 클리어할 수 있습니다.  
처음부터 큰 규모의 게임을 만들기보다는, Unity의 핵심 기능을 하나씩 익히면서 작은 완성작을 만드는 데 집중했습니다.

---

## 1. 프로젝트 목표

이 프로젝트를 진행한 목적은 다음과 같습니다.

- Unity 프로젝트 생성부터 기본 사용법 익히기
- 2D 오브젝트와 물리 시스템 이해하기
- C# 스크립트를 이용해 플레이어 조작 구현하기
- 충돌 판정과 트리거 이벤트 이해하기
- UI를 이용한 점수 및 게임 상태 표시하기
- 게임오버 / 클리어 / 재시작 흐름 구현하기

처음 Unity를 접하는 입장에서, 단순히 예제를 따라치는 것이 아니라  
기능을 하나씩 추가해가며 “게임이 만들어지는 과정” 자체를 경험하는 데 의미를 두었습니다.

---

## 2. 프로젝트 개요

### 게임 설명
- 플레이어는 좌우 이동과 점프를 할 수 있습니다.
- 장애물에 닿으면 게임오버가 됩니다.
- 코인을 획득하면 점수가 증가합니다.
- 목표 점수에 도달하면 게임 클리어가 됩니다.
- 게임오버 또는 클리어 상태에서 `R` 키를 눌러 다시 시작할 수 있습니다.

### 조작 방법
- 이동: `A`, `D` 또는 방향키 `←`, `→`
- 점프: `Space`
- 재시작: `R`

---

## 3. 구현한 기능

### 3.1 플레이어 이동
플레이어가 좌우 방향키 입력을 받아 이동하도록 구현했습니다.

학습한 내용:
- `Input.GetAxisRaw("Horizontal")`
- `Rigidbody2D`
- `velocity`를 이용한 이동 처리

처음에는 `transform.position`으로 이동을 처리했지만,  
이후 물리와 자연스럽게 동작하도록 `Rigidbody2D`의 속도를 이용해 이동하도록 수정했습니다.

---

### 3.2 점프 기능
플레이어가 스페이스바 입력으로 점프할 수 있도록 구현했습니다.

학습한 내용:
- 점프를 위한 수직 속도 설정
- `Rigidbody2D.velocity`
- 중력(`Gravity Scale`) 설정
- 회전 방지를 위한 `Freeze Rotation Z`

처음 점프 구현 시, 물리 설정과 회전 문제 때문에 플레이어가 어색하게 동작할 수 있었는데,  
RigidBody 설정을 조정하며 자연스러운 동작을 만들었습니다.

---

### 3.3 바닥 판정
공중에서 무한 점프가 되지 않도록, 바닥에 닿아 있을 때만 점프할 수 있도록 구현했습니다.

학습한 내용:
- `GroundCheck` 빈 오브젝트 생성
- `Layer`를 이용한 바닥 구분
- `Physics2D.OverlapCircle()` 사용
- `LayerMask`를 통한 특정 레이어 감지

이 기능을 통해 단순한 점프가 아니라,  
실제로 바닥 판정이 있는 점프 시스템을 구현할 수 있었습니다.

---

### 3.4 장애물 충돌 및 게임오버
장애물과 충돌했을 때 게임이 종료되도록 구현했습니다.

학습한 내용:
- `Tag`를 활용한 장애물 구분
- `OnCollisionEnter2D()` 사용
- 게임 상태를 관리하는 `GameManager` 구조

장애물에 닿았을 때 단순히 플레이어가 멈추는 것이 아니라,  
게임오버 텍스트를 표시하고 게임을 정지시키는 흐름까지 구현했습니다.

---

### 3.5 코인 획득 및 점수 시스템
코인을 획득하면 점수가 올라가도록 구현했습니다.

학습한 내용:
- `Is Trigger`를 사용한 트리거 충돌
- `OnTriggerEnter2D()` 사용
- 점수 증가 처리
- 획득 후 `Destroy(gameObject)`로 코인 제거

코인은 장애물처럼 물리적으로 막는 오브젝트가 아니라,  
플레이어가 닿았을 때 사라지고 점수를 올려주는 역할로 구현했습니다.

---

### 3.6 UI 표시
점수와 게임 상태를 플레이어가 바로 확인할 수 있도록 UI를 추가했습니다.

학습한 내용:
- `TextMeshPro`
- `ScoreText`
- `GameOverText`
- `ClearText`

단순히 기능만 동작하는 것이 아니라,  
게임의 현재 상태를 사용자에게 보여주는 것이 중요하다는 점을 배웠습니다.

---

### 3.7 게임 클리어
목표 점수에 도달하면 게임 클리어가 되도록 구현했습니다.

학습한 내용:
- 목표 점수 설정
- 현재 점수 비교
- 게임 클리어 상태 관리
- 클리어 시 UI 표시 및 게임 정지

이 기능을 통해 게임에 “목표”를 추가할 수 있었고,  
단순한 테스트 프로젝트가 아니라 최소한의 게임 구조를 갖추게 되었습니다.

---

### 3.8 재시작 기능
게임오버 또는 클리어 상태에서 `R` 키로 현재 씬을 다시 불러오도록 구현했습니다.

학습한 내용:
- `SceneManager.LoadScene()`
- 현재 씬 이름 가져오기
- `Time.timeScale` 조절

게임을 다시 실행하기 위해 에디터에서 Play를 껐다 켜지 않아도 되도록 하여,  
테스트 효율을 높일 수 있었습니다.
